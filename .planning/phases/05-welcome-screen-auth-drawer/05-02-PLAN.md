Completely rewrite `app/(onboarding)/welcome.tsx`. This is a standalone full-screen layout -- do NOT use OnboardingLayout (per user decision: "too different from other onboarding screens").

**Screen structure (top to bottom):**
1. Root `View` with `flex: 1`, `backgroundColor: semantic.bgPage`
2. BirdMosaic in top ~55% of screen (use `Dimensions.get('window').height * 0.55` for height prop)
3. White content card in bottom ~45% with:
   - "Welcome to Birda" heading (typography.h2, semantic.textPrimary)
   - Description text: "Discover, identify, and log birds around you" (typography.bodySmall, semantic.textSecondary)
   - "Create Account" button (primary, full width) -- opens auth drawer in 'signup' mode
   - "Log in" button (ghost variant, full width) -- opens auth drawer in 'login' mode
   - Both buttons disabled (opacity 0.5, not pressable) until termsAccepted is true
   - Terms checkbox row below buttons (same pattern as current welcome.tsx: Pressable with checkbox View + label text)
4. AuthDrawer rendered at the end (inside the root View, outside the content card -- BottomSheet uses Portal-like rendering)

**State management:**
- `termsAccepted` and `setTermsAccepted` from `useOnboardingStore`
- `const [drawerMode, setDrawerMode] = useState<'login' | 'signup' | null>(null)` -- tracks which button was tapped
- `const sheetRef = useRef<BottomSheet>(null)` -- controls the bottom sheet
- `const shouldNavigate = useRef(false)` -- flag for dismiss-then-navigate pattern

**Button handlers:**
- `handleCreateAccount`: if termsAccepted, set drawerMode to 'signup', call `sheetRef.current?.expand()`
- `handleLogin`: if termsAccepted, set drawerMode to 'login', call `sheetRef.current?.expand()`

**Auth selection flow (dismiss-then-navigate per user decision):**
- `onSelectOption` callback passed to AuthDrawer: sets `shouldNavigate.current = true`, calls `sheetRef.current?.close()`
- Add `onChange` prop to AuthDrawer (or handle in AuthDrawer internally): when sheet index becomes -1 (closed) AND `shouldNavigate.current` is true, call `router.push('/(onboarding)/ai-bird-id')` and reset `shouldNavigate.current = false`
- NOTE: The onChange logic should be in welcome.tsx, not inside AuthDrawer. Pass an `onClose` or use BottomSheet's `onChange` callback. If AuthDrawer doesn't expose onChange, add it as a prop: `onChange?: (index: number) => void` and pass through to BottomSheet.

**Safe area:**
- Use `useSafeAreaInsets()` for `bottom` padding on the content card (home indicator clearance)
- The mosaic does NOT need safe area -- it's decorative and crops at edges

**Content card styling:**
- `paddingHorizontal: spacing['6']` (24px)
- `paddingTop: spacing['5']` (20px)
- Buttons container with `gap: spacing['1']` (4px) between Create Account and Log in
- Use the app's `Button` component from `@/components/ui/Button` for Create Account (primary) and Log in (ghost variant)
- Wrap buttons in a View with opacity animation or simple `opacity` style based on termsAccepted
- Add `disabled` prop to both Button components when !termsAccepted

**Checkbox row (preserve from current welcome.tsx):**
- Same Pressable with checkbox View pattern
- Checkbox shows checkmark when termsAccepted
- Label: "I agree to the Terms of Service and Privacy Policy"
- Use semantic tokens for colors (actionPrimary for checked bg, borderDefault for unchecked border, textInverse for checkmark)

**Conventions:**
- `export default function WelcomeScreen()`
- `StyleSheet.create()` at bottom
- `@/` alias for all src imports
- `borderCurve: 'continuous'` on checkbox borderRadius
- Ternary with `null` for conditional rendering
- No `&&` short-circuit rendering
