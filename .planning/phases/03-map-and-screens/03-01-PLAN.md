   - Remove the `getVisibleBirds` function (replaced by Supercluster)
   - Add imports:
     - `import Supercluster from 'supercluster'`

     - `import { BirdInfoCard } from '@/components/map/BirdInfoCard'`
     - `import Ionicons from '@expo/vector-icons/Ionicons'`
     - `import { useRef, useState, useMemo } from 'react'` (add useRef, useMemo)
     - Keep existing imports for MapView, Marker, Region, etc.

   - **Supercluster setup (outside component or useMemo):**
     - Convert birds to GeoJSON points: `birds.map(bird => ({ type: 'Feature' as const, geometry: { type: 'Point' as const, coordinates: [bird.longitude, bird.latitude] }, properties: bird }))`
     - Create and load Supercluster index with config: `{ radius: 60, maxZoom: 16, minZoom: 0, minPoints: 2 }`
     - IMPORTANT: GeoJSON uses [longitude, latitude] order

   - **getClustersForRegion helper:**
     - Takes Region, computes bounding box: west = longitude - longitudeDelta/2, south = latitude - latitudeDelta/2, east = longitude + longitudeDelta/2, north = latitude + latitudeDelta/2
     - Computes zoom: `Math.round(Math.log2(360 / region.longitudeDelta))`
     - Returns `clusterIndex.getClusters(bbox, zoom)`

   - **Component changes:**
     - Add `const mapRef = useRef<MapView>(null)`
     - Replace `visibleBirds` with cluster state derived from `getClustersForRegion(region)`
     - In `onRegionChangeComplete`, update region state and recompute clusters
     - Use `useMemo` to initialize the Supercluster index once

   - **Marker rendering:**
     - Map over clusters array. For each feature:
       - If `feature.properties.cluster` is true: render a cluster marker
         - `<Marker>` with `tracksViewChanges={false}`, `anchor={{ x: 0.5, y: 0.5 }}`
         - Custom View child: 36px solid Birda blue circle (`semantic.markerCluster`), white count text (`feature.properties.point_count`), `borderCurve: 'continuous'`
         - `onPress`: call `handleClusterPress(feature.properties.cluster_id, coordinates)`
       - If not a cluster: render `<BirdMarker>` with `bird={feature.properties}` and `onPress={setSelectedBird}`
     - Key for clusters: `cluster-${feature.properties.cluster_id}`, for birds: `bird-${feature.properties.id}`

   - **handleClusterPress function:**
     - Get expansion zoom: `clusterIndex.getClusterExpansionZoom(clusterId)`
     - Compute deltas: `latitudeDelta = 360 / Math.pow(2, expansionZoom + 1)`, same for longitude
     - Animate: `mapRef.current?.animateToRegion({ latitude, longitude, latitudeDelta, longitudeDelta }, 350)`

   - **MapView configuration:**
     - Add `ref={mapRef}`
     - Add `mapType="mutedStandard"`
     - Add `showsPointsOfInterest={false}`
     - Add `pointOfInterestFilter={[]}` (belt-and-suspenders for POI bug)
     - Add `showsBuildings={false}`
     - Keep `showsUserLocation`, `showsCompass={false}`

   - **Floating UI icon replacement:**
     - Profile button (top-left): Replace emoji with `<Ionicons name="person-circle-outline" size={24} color={semantic.textPrimary} />`
     - Community button (top-right): Replace emoji with `<Ionicons name="people-outline" size={22} color={semantic.textPrimary} />`
     - Notification button (top-right): Replace emoji with `<Ionicons name="notifications-outline" size={22} color={semantic.textPrimary} />`. Keep the red dot badge.
     - Capture button (bottom): Replace emoji text with an HStack of `<Ionicons name="camera-outline" size={18} color={semantic.textInput} />` + `<Text>Capture</Text>` with `flexDirection: 'row'` and `gap: 6`

   - **Keep:** ErrorBoundary export, selectedBird state and BirdInfoCard rendering, all existing styles that still apply. Remove styles for inline components that were extracted. Add new styles for cluster markers and bottom button icon layout.
