**Step 2: Create `src/data/imageManifest.ts`.**

This is the single source of truth for ALL require() references (per user decision). Define types and export grouped constants:

```typescript
import { ImageSource } from 'expo-image'

type AssetStatus = 'exists' | 'placeholder'

interface ImageAsset {
  source: ImageSource
  filename: string
  width: number
  height: number
  location: string
  status: AssetStatus
}
```

Export these grouped constants:

**ONBOARDING_IMAGES** — A Record&lt;string, ImageSource&gt; for quick access by screen name. Keys: 'ai-bird-id', 'discover', 'goals', 'green-time', 'community', 'mailing-list', 'reminders'. Values: the require() calls pointing to src/assets/onboarding/{key}.png.

**AVATAR_IMAGES** — A Record&lt;string, ImageSource&gt; keyed by 'default', 'new', 'garden', 'intermediate', 'expert'. Values: require() calls to src/assets/avatar-default.png (for 'default') and src/assets/avatars/{key}.png (for others).

**PAYWALL_HERO** — A single ImageSource: `require('@/assets/paywall-hero.png')`.

**BIRD_IMAGES** — A Record&lt;string, ImageSource&gt; keyed by bird id, referencing the 20 require() calls from src/assets/birds/. All bird images already exist from Plan 01. Use the same kebab-case filenames from Plan 01.

**ONBOARDING_MANIFEST** — Array of ImageAsset entries for onboarding images. Status: 'placeholder'. Dimensions: ~390 x 220.

**AVATAR_MANIFEST** — Array of ImageAsset entries for avatar images. Status: 'placeholder'. Dimensions: 120 x 120.

**PAYWALL_MANIFEST** — Single ImageAsset for paywall hero. Status: 'placeholder'. Dimensions: ~390 x 280.

**BIRD_MANIFEST** — Array of ImageAsset entries for birds. Status: 'exists'. Dimensions: 320 x 240.

**IMAGE_MANIFEST** — Combined array of all ImageAsset entries for easy iteration/reporting.

Export all quick-access groups (ONBOARDING_IMAGES, AVATAR_IMAGES, PAYWALL_HERO, BIRD_IMAGES) and all manifest arrays as named exports. Follow project conventions: named exports, no default export, camelCase file name.
