Create `src/data/imageManifest.ts` as a central TypeScript registry of all image assets in the project. This file serves two purposes: (1) single source of truth for require() references, and (2) documentation of what assets exist vs what are placeholders.

Define types:
```typescript
import { ImageSource } from 'expo-image'

type AssetStatus = 'exists' | 'placeholder'

interface ImageAsset {
  source: ImageSource
  filename: string
  width: number
  height: number
  location: string        // where in the app this is used
  status: AssetStatus
}
```

Export grouped constants:

**BIRD_IMAGES** — Record keyed by bird id (string), referencing the 20 require() calls from src/assets/birds/. All status: 'exists'. Dimensions: 320x width (Wikipedia thumbnails), estimate height proportionally or use 240. Location descriptions like "Welcome mosaic, Map markers, Bird detail".

**ONBOARDING_IMAGES** — Record keyed by screen name. All 7 onboarding illustration screens. Status: 'placeholder'. Expected dimensions from the illustration style (typically full-width ~390 x height from style, e.g., 200-250). Location: "Onboarding: {screen name} hero".

**AVATAR_IMAGES** — Record keyed by skill level (new, garden, intermediate, expert) plus 'default'. Status: 'placeholder'. Dimensions: 120x120 (from avatar circle style). Location: "Onboarding: name screen" / "Onboarding: birding journey".

**PAYWALL_HERO** — Single ImageAsset. Status: 'placeholder'. Dimensions: full-width x 35% screen height. Location: "Paywall hero zone".

**IMAGE_MANIFEST** — Combined array of all ImageAsset entries for easy iteration/reporting.

Export all groups as named exports. Follow project conventions: named exports, no default export, camelCase file name.
