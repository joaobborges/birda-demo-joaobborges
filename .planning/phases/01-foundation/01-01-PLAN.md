  <action>
Modify `app/_layout.tsx` to hold the native splash screen until the Zustand onboarding store finishes rehydrating from AsyncStorage.

Implementation (follow the exact pattern from RESEARCH.md):
1. Import `* as SplashScreen from 'expo-splash-screen'`
2. Call `SplashScreen.preventAutoHideAsync()` at MODULE SCOPE (top-level, before the component function) -- this is critical, it must run before the first render
3. Add `useState` initialized with `useOnboardingStore.persist.hasHydrated()` to catch already-hydrated case
4. Add `useEffect` that subscribes to `useOnboardingStore.persist.onFinishHydration()` and sets `isHydrated = true`, returns the unsubscribe function
5. Add `useEffect` that calls `SplashScreen.hideAsync()` when `isHydrated` becomes true
6. Return `null` while `!isHydrated` -- this prevents the Stack navigator from rendering, which prevents Expo Router from auto-hiding the splash screen
7. Keep existing GestureHandlerRootView, StatusBar, Stack, and DevPanel rendering unchanged

The existing `app/index.tsx` Redirect logic already handles routing to `/(main)` vs `/(onboarding)/welcome` based on `completed` state -- no changes needed there.
