For each of the 10 onboarding screen files, perform the following systematic replacement. Read each file first, then apply changes.

**Font replacements** -- replace all hardcoded fontSize/fontWeight/fontFamily combinations with typography token spreads:
- `fontSize: 28-30, fontWeight: '700'` -> `...typography.h2` (30px) or `...typography.h3` (26px) -- pick closest
- `fontSize: 24, fontWeight: '700'` -> `...typography.h3` (26px) -- closest match
- `fontSize: 16, fontWeight: '400'` -> `...typography.bodySmall` (16px)
- `fontSize: 16` (body text) -> `...typography.bodySmall` (16px)
- `fontSize: 14` -> `...typography.caption` (14px)
- `fontSize: 12` -> add `fontFamily: fontWeights.regular, fontSize: 12` (no exact token, use closest)
- `fontSize: 15` -> `...typography.bodySmall` adjusted or `fontFamily: fontWeights.regular, fontSize: 15`
- `fontSize: 30, fontWeight: '700'` (welcome.tsx heading) -> `...typography.h2`
- Remove standalone `fontWeight` properties after replacing with typography tokens (weight is in the family name)
- When a fontSize doesn't exactly match any token, pick the CLOSEST token. If the difference would look wrong (e.g., token is 26px but design needs 14px), use `fontFamily: fontWeights.xxx` with the literal fontSize.
- Import: `import { typography, fontWeights } from '@/theme/typography'`

**Spacing replacements** -- replace hardcoded margin/padding/gap values with spacing tokens:
- `8` -> `spacing['2']`
- `12` -> `spacing['3']`
- `16` -> `spacing['4']`
- `20` -> `spacing['5']`
- `24` -> `spacing['6']`
- `32` -> `spacing['8']`
- `40` -> `spacing['10']`
- Values that don't map to tokens (e.g., 6, 10, 14, 18) -> keep as literal with a comment `// no exact token`
- Import: `import { spacing } from '@/theme/spacing'`

**Color replacements:**
- `'#FFFFFF'` in welcome.tsx checkmark -> `semantic.textInverse` or `colors.neutral0`
- Any other hardcoded hex values -> map to closest semantic token (per user decision: don't add new tokens)
