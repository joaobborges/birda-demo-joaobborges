  <name>Task 1: Fix splash screen config and runtime font loading</name>
  <files>app.json, app/_layout.tsx</files>
  <action>
Two fixes in one task because they are coupled -- splash hide must wait for fonts.

**app.json changes:**
1. Remove the deprecated top-level `splash` key (lines 11-15). It is ignored in SDK 55.
2. Replace the bare `"expo-splash-screen"` plugin string (line 47) with a configured array:
```json
["expo-splash-screen", {
  "image": "./src/assets/bird-logo.png",
  "resizeMode": "contain",
  "backgroundColor": "#ffffff"
}]
```
This ensures the splash screen plugin actually picks up the Birda logo.

**app/_layout.tsx changes:**
1. Add import: `import { useFonts, Rubik_300Light, Rubik_400Regular, Rubik_500Medium, Rubik_600SemiBold, Rubik_700Bold } from '@expo-google-fonts/rubik'`
2. Inside `RootLayout`, add the useFonts hook BEFORE the existing hydration state:
```typescript
const [fontsLoaded, fontError] = useFonts({
  Rubik_300Light,
  Rubik_400Regular,
  Rubik_500Medium,
  Rubik_600SemiBold,
  Rubik_700Bold,
})
```
3. Change the splash-hide useEffect to gate on BOTH `isHydrated` AND `(fontsLoaded || fontError)`:
```typescript
useEffect(() => {
  if (isHydrated && (fontsLoaded || fontError)) {
    const timer = setTimeout(() => {
      SplashScreen.hideAsync()
    }, 500)
    return () => clearTimeout(timer)
  }
}, [isHydrated, fontsLoaded, fontError])
```
4. Change the early return guard to also check fonts:
```typescript
if (!isHydrated || (!fontsLoaded && !fontError)) {
  return null
}
```

The `fontError` fallback ensures the app still loads even if fonts fail (graceful degradation to system font). The 500ms timer on splash hide preserves the brief splash screen visibility.

Do NOT remove the expo-font config plugin -- it is still needed for production/EAS builds.

Fix the two compounding issues causing clipping and empty space on reminders and mailing-list screens.

**In OnboardingLayout.tsx:**

1. Add a fallback safe area `paddingTop` when NEITHER `illustration` NOR `header` is provided. After the existing `header`-only branch (line 30-32), add an else branch that renders a spacer View:
```typescript
) : (
  <View style={{ paddingTop: top + spacing['5'] }} />
)}
```
This ensures content never starts behind the status bar/notch, regardless of which props are passed.

2. Fix the vertical content distribution. Change `scrollContent` style from `flexGrow: 1` to `flexGrow: 0`. The `flexGrow: 1` causes the scroll content area to stretch and fill the entire screen when there is no illustration, creating the large empty space. With `flexGrow: 0`, the content sizes to its natural height.

3. To keep content vertically centered for short-content screens (like reminders), add `justifyContent: 'center'` and keep `flexGrow: 1` ONLY on the `scroll` style (which is already `flex: 1`). Actually, the better approach: change `scrollContent` to use `flexGrow: 1` with `justifyContent: 'center'` so content is vertically centered in the available space rather than pinned to the top.

Updated scrollContent style:
```typescript
scrollContent: {
  flexGrow: 1,
  justifyContent: 'center',
  paddingHorizontal: spacing['6'],
}
```

This combination means: the scroll content area fills available space (flexGrow:1) AND centers its children vertically (justifyContent:'center'). For screens WITH illustration, the illustration takes ~50% of the screen so the scroll area is smaller and centering works naturally. For screens WITHOUT illustration, the content centers in the full scroll area instead of pinning to the top with empty space below.

Do NOT modify reminders.tsx or mailing-list.tsx -- the fix is entirely in OnboardingLayout so it benefits all current and future screens.
