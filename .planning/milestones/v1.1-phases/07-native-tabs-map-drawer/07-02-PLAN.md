<task type="auto">
  <name>Task 2: Add triple-tap gesture to root layout to trigger DevPanel</name>
  <files>app/_layout.tsx</files>
  <action>
In `app/_layout.tsx`:

1. **Add imports:**
```tsx
import { useState } from 'react' // already imported
import { View } from 'react-native'
import { GestureDetector, Gesture } from 'react-native-gesture-handler'
```

2. **Add state for dev panel visibility** inside `RootLayout`:
```tsx
const [devPanelVisible, setDevPanelVisible] = useState(false)
```

3. **Create the triple-tap gesture** inside the component (must be inside to access `setDevPanelVisible`):
```tsx
const tripleTap = Gesture.Tap()
  .numberOfTaps(3)
  .maxDuration(500)
  .runOnJS(true)
  .onEnd(() => setDevPanelVisible(true))
```

IMPORTANT: Use `.runOnJS(true)` because `setState` must run on the JS thread, not the UI thread. Without this, the app will crash.

4. **Wrap the Stack in a GestureDetector + View** (the View is required as GestureDetector needs a native view child):
```tsx
<GestureHandlerRootView style={{ flex: 1 }}>
  <StatusBar style="dark" />
  <GestureDetector gesture={tripleTap}>
    <View style={{ flex: 1 }}>
      <Stack screenOptions={{ headerShown: false }} />
    </View>
  </GestureDetector>
  {DevPanel ? (
    <DevPanel
      visible={devPanelVisible}
      onClose={() => setDevPanelVisible(false)}
    />
  ) : null}
</GestureHandlerRootView>
```

Key points:
- `GestureDetector` MUST be inside `GestureHandlerRootView` (it already is)
- DevPanel renders OUTSIDE the GestureDetector so the Modal is not affected by the tap gesture
- The triple-tap should not conflict with MapView gestures because tap gestures with 3 taps have a very different activation pattern from pan/pinch gestures
- Keep the `__DEV__` conditional loading of DevPanel
