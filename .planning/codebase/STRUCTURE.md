## Directory Purposes

**app/:**
- Purpose: All screens and navigation structure
- Contains: Route files (`.tsx`), layout files (`_layout.tsx`), route groups (`(main)`, `(onboarding)`)
- Key files: `app/index.tsx` (root router), `app/(main)/index.tsx` (map home screen)
- Convention: Each file becomes a route. `_layout.tsx` defines the navigator for its directory. Groups in parentheses `()` share a layout without affecting URL.

**src/components/:**
- Purpose: Reusable components organized by feature domain
- Contains: Subdirectories per domain (`ui/`, `onboarding/`, `dev/`)
- Key files: `src/components/ui/Button.tsx` (used across all screens)

**src/data/:**
- Purpose: Static data constants and their TypeScript interfaces
- Contains: Data files exporting typed arrays
- Key files: `src/data/birds.ts` (bird catalog with geolocation data)

**src/stores/:**
- Purpose: Global state management
- Contains: Zustand store definitions with persistence config
- Key files: `src/stores/onboarding.ts`

**assets/:**
- Purpose: Static image assets for app icons and splash screens
- Contains: PNG files referenced in `app.json`

**ios/:**
- Purpose: Native iOS project generated by Expo prebuild
- Contains: Xcode project, Pods, build artifacts
- Generated: Yes (via `expo prebuild`)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `app/_layout.tsx`: Root layout wrapping all screens
- `app/index.tsx`: Initial route decision (onboarding vs main)
- `package.json` `"main": "expo-router/entry"`: Expo Router entry point

**Configuration:**
- `app.json`: Expo app config (name, icons, splash, plugins, bundle IDs)
- `eas.json`: EAS Build profiles (development, preview, production)
- `tsconfig.json`: TypeScript config with `@/*` path alias mapping to `./src/*`
- `metro.config.js`: Metro bundler config with experimental import support

**Core Logic:**
- `src/stores/onboarding.ts`: Onboarding state machine (the only global store)
- `src/data/birds.ts`: Bird data model and static dataset
- `app/(main)/index.tsx`: Map rendering and bird visibility logic

**Reusable Components:**
- `src/components/ui/Button.tsx`: Primary button component (3 variants)
- `src/components/onboarding/ProgressDots.tsx`: Step indicator

## Naming Conventions

**Files:**
- Screen files: `kebab-case.tsx` (e.g., `skill-level.tsx`)
- Component files: `PascalCase.tsx` (e.g., `Button.tsx`, `DevPanel.tsx`, `ProgressDots.tsx`)
- Data/store files: `camelCase.ts` (e.g., `birds.ts`, `onboarding.ts`)
- Layout files: `_layout.tsx` (Expo Router convention)

**Directories:**
- Route groups: parenthesized lowercase `(main)`, `(onboarding)`
- Feature directories: lowercase (`ui/`, `dev/`, `onboarding/`)
- Top-level: lowercase (`app/`, `src/`, `assets/`)

**Exports:**
- Screen components: `export default function ScreenNameScreen()` (e.g., `MapScreen`, `WelcomeScreen`)
- Shared components: Named exports (e.g., `export function Button()`, `export function ProgressDots()`)
- Stores: Named export of hook (e.g., `export const useOnboardingStore`)
- Data: Named exports of types and constants (e.g., `export interface Bird`, `export const birds`)

## Where to Add New Code

**New Screen:**
- Main app screen: `app/(main)/new-screen.tsx` -- add `<Stack.Screen>` entry in `app/(main)/_layout.tsx` if custom options needed
- Onboarding step: `app/(onboarding)/new-step.tsx` -- update navigation chain in adjacent step files

**New Reusable Component:**
- UI primitive (buttons, inputs, cards): `src/components/ui/ComponentName.tsx`
- Feature-specific component: `src/components/{feature}/ComponentName.tsx`
- Use named exports, not default exports

**New Data Model:**
- Static data: `src/data/model-name.ts` with interface + const array
- Example: follow pattern in `src/data/birds.ts`

**New State Store:**
- Zustand store: `src/stores/store-name.ts`
- Follow pattern in `src/stores/onboarding.ts`: interface, `create()` with `persist()` middleware
- Export as `use{Name}Store` hook

**New Utility/Helper:**
- Create `src/utils/` directory (does not exist yet)
- Or `src/lib/` for third-party wrappers

**New Assets:**
- Static images: `assets/`

## Special Directories

**.planning/:**

- Generated: Yes (by GSD tooling)
